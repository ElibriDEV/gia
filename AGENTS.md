# AGENTS.md

## Кратко о проекте

`gia` — статический HTML/PWA-тренажер для тестирования. Стартовое меню делит каждый раздел на билеты максимум по 16 вопросов, дает марафон по разделу и общий марафон по всем разделам.

## Основные директории

- Корневой каталог — статический тренажер без сборщика и backend.
- `.idea/` — служебные файлы IDE, игнорируются git.

## Ключевые файлы

- `index.html` — меню выбора раздела и билета.
- `index-page.js` — построение карточек билетов по категориям из `quiz-categories.js`.
- `quiz-categories.js` — карта пользовательских категорий и фильтры поверх `embedded-data.js`.
- `c-basic.html` — тест по специфичным вопросам C из `C.csv`.
- `c.html` — тест по C++, общим темам C/C++, инструментам разработки и программной инженерии из `C.csv`.
- `security.html` — тест по защите информации (`Защита_информации.csv`).
- `networks.html` — тест по сетям (`Сети_норм.csv`, включая импортированные карточки из Wellemo).
- `marathon.html` — общий марафон по всем разделам.
- `quiz-page.js` — общая бизнес-логика карточек, билетов, марафонов, перемешивания вариантов и подсчета результата.
- `embedded-data.js` — встроенные данные вопросов, правильных ответов и ручных дистракторов.
- `manifest.webmanifest` — manifest для установки PWA на Android.
- `sw.js` — service worker, кэширует статические ресурсы для офлайн-режима.
- `pwa.js` — регистрация service worker в браузере.
- `icon.svg` — иконка PWA.
- `styles.css` — общие стили.
- `README.md` — пользовательская документация.
- `.gitignore` — исключения для зависимостей, IDE-файлов, логов, локальных env и временных артефактов.

## Карта модулей

- Каждая тестовая HTML-страница задает `window.QUIZ_FILE`.
- Страницы с фильтром дополнительно задают `window.QUIZ_CATEGORY`.
- Затем подключаются `embedded-data.js`, `quiz-categories.js` и `quiz-page.js`.
- `quiz-categories.js` публикует `window.QUIZ_CATEGORIES` и `window.getQuizCategoryQuestions`.
- `quiz-page.js` берет пул через категорию, если она задана; иначе использует `window.EMBEDDED_QUIZ_DATA[window.QUIZ_FILE]`.
- Номер билета передается в тестовую страницу через query-параметр `ticket`.
- Режим марафона передается через query-параметр `mode=marathon`.
- Общий марафон передается через query-параметр `mode=all-marathon`.
- `index-page.js` рассчитывает количество билетов как `Math.ceil(questions.length / 16)` по отфильтрованной категории.
- Каждый объект вопроса должен содержать `question`, `answer`, `topic` и `distractors` из 3 строк.
- PWA подключается на всех HTML-страницах через `manifest.webmanifest` и `pwa.js`.
- `sw.js` должен кэшировать все файлы, необходимые для офлайн-работы.
- `C.csv` разделен на две категории в `quiz-categories.js`: `Си` и `C++ и общее C/C++`.
- В `Сети_норм.csv` есть импортированная тема `Сети_МС Юдоу`; правильные ответы в ней взяты из зеленого выделения в HTML-выгрузке Wellemo.

## Бизнес-логика

В `quiz-page.js`:

- `getTicketNumber` и `getTicketPool` — выбор билета по URL и срез вопросов максимум по 16.
- `isMarathonMode` — включение режима всех вопросов через `mode=marathon`.
- `isAllMarathonMode` и `getAllQuestions` — общий марафон из всех категорий через `mode=all-marathon`.
- `getCurrentCategory`, `getCurrentPool` — выбор категории и отфильтрованного пула вопросов.
- `updatePageHeader` — замена статического заголовка предметной страницы для специальных режимов.
- `buildTicket` — формирование раунда из вопросов выбранного билета.
- `shuffle` — перемешивание вопросов билета и вариантов ответа.
- `getDistractors` — проверка и выдача 3 заранее заданных неверных ответов.
- `startQuiz` — показ карточек по одной и итоговая оценка.

## Конфигурация и окружение

- Backend и env-переменные не требуются.
- Проект рассчитан на GitHub Pages или другой статический хостинг.

## Тесты и проверки

- Автотесты отсутствуют.
- Ручная проверка: открыть страницу, пройти раунд, проверить итог.
- PWA-проверка: открыть опубликованную HTTP/HTTPS-страницу, убедиться, что manifest и service worker загружаются.
- Проверка категорий: выполнить Node-команду из README.md.

## Правила репозитория

- Все текстовые файлы сохранять в UTF-8.
- После значимых изменений обновлять `README.md` и `AGENTS.md`.
- Не переводить проект на серверный режим без запроса пользователя.
- Не возвращать генерацию дистракторов в рантайм без отдельного запроса пользователя.
- При изменении размера билета синхронизировать `ticketSize` в `index-page.js` и `quiz-page.js`.
- При добавлении новых статических файлов обновлять список `appShell` в `sw.js`.
- При изменении состава вопросов `C.csv` перепроверить список `ordinaryCQuestionIndexes` в `quiz-categories.js`.

## Чувствительные места

- Кодировка кириллицы в `embedded-data.js`.
- Импорт карточек из Wellemo: не полагаться на `selectedAnswer`, если правильный вариант задан зеленым выделением в HTML.
- Список фильтра специфичных вопросов C в `quiz-categories.js`: он завязан на текущий порядок вопросов в `C.csv`.
- Корректность парсинга и формата встроенных данных.
- Качество дистракторов: не допускать слишком очевидных неверных ответов.
- Полнота поля `distractors`: у каждого вопроса должны быть ровно 3 уникальные строки, не совпадающие с правильным ответом.
