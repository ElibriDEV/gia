# GIA Quiz

Статический тренажер тестов с отдельными HTML-страницами по разделам вопросов.
Проект также работает как PWA-приложение для Android: его можно открыть в Chrome и установить на главный экран.

## Страницы

- `c.html` — объединенный тест по C/C++ из `C.csv`
- `csharp.html` — тест по C# из встроенного набора `CSharp.csv`
- `security.html` — тест по `Защита_информации.csv`
- `networks.html` — тест по `Сети_норм.csv`, включая импортированные карточки из выгрузки Wellemo
- `marathon.html` — общий марафон по всем разделам
- `index.html` — стартовое меню выбора предмета и билета

## Как это работает

- Каждый предмет разбит на билеты максимум по 16 вопросов.
- На стартовой странице есть поиск по формулировкам вопросов: результат показывает раздел, номер билета и ссылку на него.
- Билет или режим марафона выбирается на стартовой странице.
- Вопросы выбранного билета показываются по одной карточке в случайном порядке.
- В режиме марафона в один раунд попадают все вопросы выбранного предмета.
- В общем марафоне в один раунд попадают все вопросы из всех разделов вперемешку.
- На каждый вопрос дается 4 варианта:
  - 1 правильный
  - 3 заранее заданных неправильных ответа.
- Порядок вариантов ответа перемешивается при каждом запуске билета.

## Неправильные ответы

Неверные варианты хранятся прямо в `embedded-data.js` в поле `distractors` у каждого вопроса.

`quiz-page.js` больше не генерирует дистракторы при запуске. Если у вопроса не задано 3 уникальных неправильных ответа, страница считает данные некорректными.

## Структура проекта

- `index.html` — меню выбора предмета и билета.
- `index-page.js` — построение стартовых карточек билетов и поиска по вопросам по данным из `embedded-data.js`.
- `quiz-categories.js` — карта категорий поверх встроенных данных.
- `c.html`, `csharp.html`, `security.html`, `networks.html` — отдельные тестовые страницы.
- `marathon.html` — отдельная страница общего марафона без привязки к названию предмета.
- `quiz-page.js` — общая логика теста (карточки, рандом, проверка, итог).
- `manifest.webmanifest` — описание PWA-приложения для установки на Android.
- `sw.js` — service worker для офлайн-кэша.
- `pwa.js` — регистрация service worker.
- `icon.svg` — иконка приложения.
- `embedded-data.js` — встроенная база вопросов, правильных ответов и 3 неправильных ответов для каждого вопроса.
- `styles.css` — стили.

## Категории

`quiz-categories.js` публикует разделы стартового меню:

- `C/C++` — объединенный набор вопросов из `C.csv`; билеты нарезаются последовательно по порядку исходного набора, а затем вопросы выбранного билета перемешиваются при запуске раунда.
- `C#` — отдельный набор из 40 вопросов по основам C#, типам, классам, структурам, модификаторам, исключениям и небезопасному коду.
- `Защита информации` — вопросы из `Защита_информации.csv`.
- `Сети` — вопросы из `Сети_норм.csv`.

Источники вопросов подключаются как отдельные категории без фильтрации.

В сетевой категории есть дополнительная тема `Сети_МС Юдоу`: карточки импортированы из HTML-выгрузки Wellemo, где правильный вариант определялся по зеленому выделению ответа.

## Запуск

### Локальный просмотр

1. Откройте `index.html`.
2. Выберите нужный предмет и билет или режим марафона.
3. При необходимости используйте поиск по вопросам на стартовой странице, чтобы найти билет с нужной формулировкой.
4. Пройдите карточки выбранного режима, отвечая на вопросы.

Для простого просмотра сервер не требуется.

### GitHub Pages и Android PWA

1. Залейте файлы проекта в репозиторий GitHub.
2. Включите GitHub Pages для ветки `main` и папки `/root`.
3. Откройте опубликованную HTTPS-ссылку в Chrome на Android.
4. В меню Chrome выберите `Добавить на главный экран` или `Установить приложение`.

После первой загрузки основные файлы кэшируются service worker и доступны офлайн. Для установки PWA нужен HTTP/HTTPS-адрес; режим `file://` подходит только для обычного просмотра.

